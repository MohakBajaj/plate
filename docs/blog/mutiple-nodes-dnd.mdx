## 前言

在富文本编辑器中，拖拽功能往往是被忽视却又至关重要的一环。

一次好的拖拽交互，能显著提升内容整理和结构调整的效率。而一次糟糕的体验，则可能让用户彻底放弃使用。

在 Plate.js 49.0.8 之后，拖拽的手感和成功率有了显著提升。但与此同时，也暴露出了一些边缘情况的处理短板。

本文将带你深入分析此次升级带来的改变，以及未来仍需改进的方向。

## 拖拽体验的升级点

### 更稳定的拖放逻辑

在旧版本中，拖动块级元素到其他块之间时，**容易出现 drop 失败、光标位置错乱** 的情况。

在 49.0.8 中，经过一系列调整后，drop 区域的检测变得更智能，尤其是在相邻 block 高度接近时的判断更加精准，几乎不会出现拖拽失效的问题。

### 手感更顺滑

相比之前“卡顿”“要多拖几次才生效”的体验，现在的拖拽手感更贴近 Notion，能够更清晰地预览插入位置，反馈及时。

## 仍需改进的两个问题

### 问题一：在 margin-top 上 drop 会失败

目前，如果目标块上方存在明显的 `margin-top`，在视觉上用户以为可以 drop 到该区域，但实际 drop 会被忽略，导致体验断裂。

这是因为检测插入位置时使用的是 block 的实际边界，而非其可见区域。

**解决方向建议：**
可以在检测 drop 位置时引入 margin 的视觉感知，或允许 drop 区域向外扩展几个像素。

### 问题二：暂不支持多块同时拖拽

当前版本仍**仅支持单块拖拽**，这在用户需要快速移动一段内容（例如一个 H2 + 段落 + 图片）时显得很不够用。

虽然通过临时方法（如包装成一个 group）可以间接实现，但这不是理想的体验。

**理想行为：**
- 选中多个块后，一次拖拽；
- 拖动过程中高亮整个 selection；
- drop 后保留原始顺序和样式。

## 小结

Plate.js 49.0.8 的拖拽系统已经跨出关键一步，让整体体验更贴近主流富文本编辑器（如 Notion）。

但要真正做到“随心所欲”的拖拽，还需要：

- 对边缘空白区 drop 的支持；
- 多块 selection + 拖动的能力；
- 更自然的插入动画与过渡效果。

未来版本如果能继续优化这些点，Plate 的编辑体验将进一步向“所见即所得”迈进。
